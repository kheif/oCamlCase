Constructors and Exceptions

Constructor types let you define custom types with multiple alternatives, serving as the foundation for complex data structures and error handling in OCaml.

Constructor Types (Variant Types)

Constructor types let you define custom types with multiple alternatives. Think of them as tagged unions.

Key rules for variants:

  • Type names start with lowercase.
  • Constructor names start with uppercase.
  • Constructors can be used in pattern matching.
shape.ml 
type shape =
  | Circle of float
  | Square of float
  | Triangle of float * float * float

let area = function
  | Circle (r : float) -> Float.pi *. r *. r
  | Square (a : float) -> a *. a
  | Triangle ((a : float), (b : float), (c : float)) ->
      let s = (a +. b +. c) /. 2.0 in
      Float.sqrt (s *. (s -. a) *. (s -. b) *. (s -. c))
area (Triangle (3.0, 4.0, 5.0))
match shape with
| Circle r -> ...
| Square a -> ...
| Triangle (a, b, c) ->
let s = (a +. b +. c) /. 2.0
Float.sqrt ...
Triangle
3.0
4.0
5.0
Local Bindings
let a = 3.0

Result: 6.0
Step 1. We pass a variant value to the function. Notice how the tag Triangle bundles the three floats together into one value.
Step 2. The pattern match tests the tag against each branch. It skips Circle and Square, successfully matching on Triangle.
Step 3. Upon matching, OCaml automatically extracts the payload values and binds them to the local variables a, b, and c.
Step 4. The expression on the right side of the branch can now use these bindings to compute the final area.

Nullary vs Unary Constructors

Constructors can carry data or act as simple tags.

  • Nullary constructors carry no data. A type where all constructors are nullary is called an enumeration type.
  • Unary constructors carry exactly one piece of data.

OCaml has no constructors of arity two or more; tuples are used instead as a single argument, as seen in the Triangle example.

enums.ml 
type bool = false | true
type day = Monday | Tuesday | Wednesday | Thursday | Friday | Saturday | Sunday

Recursive Constructor Types

Constructor types can reference themselves, making them recursive. This is essential for representing tree like data structures such as arithmetic expressions.

expr.ml 
type var = string

type exp =
  | C of int          (* constant *)
  | V of var          (* variable *)
  | A of exp * exp    (* addition *)
  | M of exp * exp    (* multiplication *)

Computing Subexpressions (Structural Recursion)

Functions on recursive types naturally use structural recursion, meaning they recurse on the sub components of the type.

Structural recursion follows the structure of the type. Each recursive call is on a direct sub component of the input.

Important distinction

Recursion in fold_left is structural (recurses on the tail). Recursion in msort is not structural because the sublists from split are not direct sub components of the input list.

subexps.ml 
let rec subexps e = e :: (match e with
  | A (e1, e2) -> subexps e1 @ subexps e2
  | M (e1, e2) -> subexps e1 @ subexps e2
  | _ -> [])

Environments

An environment maps variable names to values. In OCaml, this is modeled as a function.

env.ml 
type env = var -> int

exception Unbound

let env = function
  | "x" -> 0
  | "y" -> 3
  | _ -> raise Unbound

Exceptions

Exceptions in OCaml are values of the built in extensible constructor type exn.

The exn type can be extended at any time with new exception constructors. Exception constructors work like normal constructors and can be used in pattern matching.

A raise expression can have any type because it never returns a value.

exceptions.ml 
exception New              (* nullary *)
exception Newer of int     (* with payload *)

let test f = try f () with
  | Newer x -> x
  | Failure _ -> -1

Execution Order

OCaml uses a right to left evaluation order for arguments.

Example: (failwith "left", (failwith "mid" + 4) + failwith "right") raises Failure "right" because the rightmost expression is evaluated first.

Practical Example: Test for Duplicates

Exceptions can be used as a control flow mechanism. Sorting can be used to detect duplicates efficiently by throwing an exception when an equality is found.

duplicate.ml 
exception Duplicate

let mask cmp p q =
  match cmp p q with
  | 0 -> raise Duplicate
  | v -> v

let test_duplicate cmp xs =
  try
    let _ = List.sort (mask cmp) xs in
    false
  with Duplicate -> true
try
List.sort (mask cmp) [1; 2; 1]
false
with Duplicate -> true
mask cmp 1 1
cmp 1 1 = 0
raise Duplicate
merge [1] [1; 2]
sort [1; 2; 1]
💥
Step 1. We establish an exception handler with try ... with and call the standard library sorting function.
Step 2. List.sort builds a deep call stack of recursive sort and merge frames. It calls our mask function to compare 1 and 1.
Step 3. cmp 1 1 returns 0. The mask function triggers raise Duplicate.
Step 4. Control immediately leaps out of the deep stack, destroying all intermediate frames. It bypasses false and lands in the with block, returning true.

Type Constructors (Parameterized Types)

Types can be parameterized to work with any element type.

parameterized.ml 
type 'a my_list = Nil | Cons of 'a * 'a my_list

This declares a type constructor my_list that produces a concrete type for every type 'a. The constructors Nil and Cons are polymorphic.

Options

The built in option type is a parameterized type.

Options are an alternative to exceptions for handling partial functions. Instead of raising an error, you return None for failure and Some value for success.

options.ml 
(* type 'a option = None | Some of 'a *)

let rec nth_opt xs n =
  if n < 0 || xs = [] then None
  else if n = 0 then Some (List.hd xs)
  else nth_opt (List.tl xs) (n - 1)

nth_opt [3; 4; 5] 3
(* Returns None instead of crashing *)
