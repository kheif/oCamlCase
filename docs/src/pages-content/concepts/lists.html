Lists

A list in OCaml is an immutable, singly-linked sequence of values of the same type. Most everyday data processing in OCaml passes through lists and the small set of functions that operate on them.

Construction

A list is either empty ([]) or a head element followed by a tail list (head :: tail). The :: operator, called cons, prepends an element to an existing list. The literal syntax [1; 2; 3] is sugar for 1 :: 2 :: 3 :: [].

All elements of a list have the same type. The type is written 'a list, where 'a is the element type. A list of integers is int list, a list of strings is string list, and an empty list is polymorphic until used.

Key point

Lists are immutable. Cons does not modify the tail; it produces a new list that shares the tail with the old one. This sharing is what makes immutable lists cheap to build.

build.ml 
let empty = []
(* empty : 'a list *)

let nums = [1; 2; 3]
(* int list *)

(* The same list with cons *)
let nums' = 1 :: 2 :: 3 :: []

(* Prepend; original list is unchanged *)
let more = 0 :: nums
(* more = [0; 1; 2; 3]
   nums  = [1; 2; 3] still *)

Pattern matching on lists

Pattern matching is the natural way to take a list apart. The two cases mirror the two ways a list can be built: empty or cons. Most list-processing functions follow the same shape: handle the empty case, then handle the cons case recursively.

The empty pattern [] matches the empty list. The cons pattern x :: xs matches any non-empty list, binding the first element to x and the rest to xs. Together they cover every possible list, and the compiler will warn if you miss either.

recurse.ml 
let rec length = function
  | [] -> 0
  | _ :: xs -> 1 + length xs

let rec sum = function
  | [] -> 0
  | x :: xs -> x + sum xs

length [10; 20; 30]   (* 3 *)
sum    [10; 20; 30]   (* 60 *)

Basic functions on lists

The standard library's List module covers the operations you reach for most often. A handful of these handle the bulk of everyday work.

list_basics.ml 
List.length [1; 2; 3]
(* 3 *)

List.rev [1; 2; 3]
(* [3; 2; 1] *)

List.append [1; 2] [3; 4]
(* [1; 2; 3; 4]. Also written [1; 2] @ [3; 4] *)

List.hd [1; 2; 3]
(* 1, raises on empty lists *)

List.tl [1; 2; 3]
(* [2; 3], raises on empty lists *)

List.nth [10; 20; 30] 1
(* 20, zero-indexed *)

Map, filter, and fold

Three higher-order functions cover most list transformations. List.map applies a function to every element. List.filter keeps the elements that satisfy a predicate. List.fold_left reduces a list to a single value by accumulating from left to right.

Together with the pipe operator, these compose into readable pipelines. Each step is one transformation, and the type of the value flowing through changes only when you map to a new element type.

Watch out

List.hd, List.tl, and List.nth raise exceptions on inputs that do not fit. For total alternatives, use the _opt variants where they exist, or pattern match yourself.

pipeline.ml 
[1; 2; 3; 4; 5]
|> List.filter (fun x -> x mod 2 = 0)
|> List.map (fun x -> x * x)
|> List.fold_left (+) 0
(* 4 + 16 = 20 *)

List.map String.length ["hi"; "there"]
(* [2; 5] *)

List.filter ((<) 0) [-1; 2; -3; 4]
(* [2; 4] *)
Good practice

Reach for map, filter, and fold_left before writing an explicit recursion. They express intent more directly and are tail-recursive where it matters.
