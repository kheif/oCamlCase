Sorting

Sorting is a fundamental operation that showcases how OCaml handles algorithms, polymorphic comparison, and recursion strategies like divide and conquer.

Insertion Sort

The idea is simple: take each element and insert it into its correct position in an already sorted list.

The insert function takes a sorted list and a value, placing it where it belongs. If the value is greater than the head, it keeps the head in place and recurses on the tail. Otherwise, it just prepends the value.

Key point

isort uses fold_left to build the sorted list element by element. It starts with an empty accumulator and inserts each element from the input list one at a time.

insertion_sort.ml 
let rec insert ys x =
  match ys with
  | y :: yr when x > y -> y :: insert yr x
  | _ -> x :: ys

let isort xs = List.fold_left insert [] xs
isort [3; 1; 4; 2]

Step 1. fold_left begins with an empty accumulator [] and the full list.
Step 2. The first element 3 is inserted into [], yielding [3].
Step 3. 1 is inserted. Because 1 < 3, it prepends 3, shifting it right visually.
Step 4. 4 is inserted. It recursively passes 1 and 3, landing at the end.
Step 5. 2 is inserted. It passes 1, but is less than 3, so it slides into the middle. The array is fully sorted.

Polymorphic Sorting

To sort different types (int, float, string), we parameterize the sort with a comparison function cmp.

polymorphic_sort.ml 
let pisort cmp =
  let rec insert ys x =
    match ys with
    | y :: yr when cmp x y > 0 -> y :: insert yr x
    | _ -> x :: ys
  in List.fold_left insert []

pisort Int.compare [5; 2; 2; 3; -2]
(* [-2; 2; 2; 3; 5] *)

pisort Float.compare [5.0; 2.0; 2.0; 3.0; -2.0]
(* [-2.0; 2.0; 2.0; 3.0; 5.0] *)

Comparison Functions

OCaml represents ordering through comparison functions of type 'a -> 'a -> int.

  • Returns a negative integer if a < b
  • Returns 0 if a = b
  • Returns a positive integer if a > b

Inverting order is done by simply swapping the arguments. This lets you sort in descending order.

invert.ml 
let invert (cmp : 'a -> 'a -> int) x y = cmp y x

pisort (invert Int.compare) [5; 2; 2; 3; -2]
(* [5; 3; 2; 2; -2] *)

Merge Sort

Merge sort uses a divide and conquer approach with three steps: split the list into two sublists, sort each recursively, and merge them back together.

split uses fold_left to alternate elements between two accumulators, distributing them evenly.

merge combines two sorted lists by comparing their heads and recursing on the remaining elements.

merge_sort.ml 
let split xs =
  List.fold_left (fun (ys,zs) x -> (zs, x::ys)) ([],[]) xs

let rec merge xs ys =
  match (xs,ys) with
  | ([],ys) -> ys
  | (xs,[]) -> xs
  | (x::xs,y::ys) ->
      if x < y then x :: merge xs (y::ys)
      else y :: merge (x::xs) ys

let rec msort xs =
  match xs with
  | [] -> []
  | [x] -> [x]
  | xs ->
      let (ys,zs) = split xs in
      merge (msort ys) (msort zs)

Step 1. We want to sort [3; 1; 4; 2]. Merge sort begins by passing it to split.
Step 2. split alternates elements, creating [3; 4] and [1; 2].
Step 3. These sublists are split again until they reach the base case: lists of length 1, which are trivially sorted.
Step 4. The recursive calls return, and merge combines the leaves back into sorted sublists. Notice the elements are now ordered.
Step 5. The final merge combines [3; 4] and [1; 2] by repeatedly taking the smallest head, yielding the fully sorted list.
