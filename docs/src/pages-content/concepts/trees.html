Trees

Trees are recursive data structures that naturally model hierarchical information. Rose trees in particular offer flexibility by allowing nodes to have any number of branches.

Rose Trees

Rose trees have a variable and unbounded number of branches per node.

Each node T holds a list of its children. This makes it incredibly expressive, allowing for structures where some nodes have no children while others have many.

tree.ml 
type tree = T of tree list

let t1 = T []
(* A single leaf / atomic tree *)

let t2 = T [t1; t1; t1]
(* A node with three leaf children *)

let t3 = T [T [t2]; t1; t2]
(* A deeper nested tree *)
Step 1. let t1 = T [] creates a single atomic tree (a leaf with no children).
Step 2. let t2 = T [t1; t1; t1] creates a new root node containing a list of three t1 children.
Step 3. let t3 = T [T [t2]; t1; t2] embeds the previous trees into an even deeper structure, demonstrating unbounded arity.

Terminology

Understanding tree terminology is essential for navigating and manipulating these structures:

  • Atomic tree: T [] (no children)
  • Compound tree: any tree with at least one child
  • Root: the top node
  • Inner nodes: non leaf nodes
  • Arity of a tree t = T [t0; ...; t_{n-1}]: the number n of its children
  • Successor trees: the children t0, ..., t_{n-1}
properties.ml 
let arity (T ts) = List.length ts

let succtree (T ts) k = List.nth ts k

Computing the Number of Nodes

The node count is computed recursively from successor trees. Each leaf contributes 1. Each inner node contributes 1 plus the sum of its children's node counts.

The approach combines three ideas:

  1. Recurse on successor trees.
  2. Use map to apply the function to each child.
  3. Use fold_left to aggregate the results.
Pattern

This function maps size over all children to get a list of counts, then folds with addition starting from 1 for the current node itself.

size.ml 
let rec size (T ts) =
  List.fold_left (+) 1 (List.map size ts)

Step 1. We want to compute the size of a tree with one root and two atomic children.
Step 2. The root calls List.map size on its children, propagating downwards.
Step 3. The leaves evaluate their base case (1) and return these values to the map function, yielding [1; 1].
Step 4. The root node receives [1; 1] and uses fold_left (+) 1 to sum the list plus its own 1, returning 3.

Step 1. We compute the size of a more detailed tree: T [T [T []; T []]; T []].
Step 2. The root maps over its two children.
Step 3. The left child maps size over its own two children.
Step 4. All three leaf nodes evaluate their base case and return 1.
Step 5. The left intermediate node receives [1; 1] and folds them plus its own 1, yielding 3.
Step 6. The root node receives [3; 1] and folds them plus its own 1, yielding 5.
